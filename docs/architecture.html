📌 Annotated reference (every numbered pin)
  1. Compute Engine VM (loadgen) — `e2-small` Debian 12, no public IP, egress via Cloud NAT. systemd unit sentinel-loadgen.service runs traffic_generator.py.
  2. Cloud Functions (chaos-inject) — flips the VM metadata sentinel-mode attribute (baseline / drift / latency / burst / off) for live demos.
  3. Pub/Sub · raw-traffic — every synthetic shopper request lands here as JSON. Consumed by the analytics path.
  4. Pub/Sub · telemetry — per-metric stream (latency_ms, image_emb_norm, review_token_count, error_rate). Source of truth for the forecaster.
  5. Pub/Sub · incidents (+ retrain · DLQ) — fan-out for actions: triage push subscription & retrain trigger; failures land in DLQ.
  6. GKE · telemetry-collector — single Deployment, batched insert_rows_json into BigQuery. ~140 rows/s sustained.
  7. GKE · forecast-detector — runs the temporal forecaster + SLO evaluator; on predicted breach it publishes a structured incident.
  8. BigQuery — datasets sentinel_dev_features & sentinel_dev_incidents. Tables: telemetry_raw, predictions, incidents.
  9. Cloud Storage — buckets …-models, …-code, …-raw-data, …-processed, …-tb-logs.
  10. Artifact Registry — Docker repo sentinel-images in us-central1; built locally by scripts/build_local.sh.
  11. Secret Manager — secrets slack-webhook-url, gemini-api-key, alert-email; read at runtime, never in code.
  12. Vertex AI Endpoints · victim-recommender — two-tower recommender served from victim:latest on n1-standard-2.
  13. Vertex AI Pipelines — KFP DAGs victim_train & forecaster_train; register_model_op auto-deploys passing models.
  14. Vertex Workbench — researcher notebook on the same VPC for ad-hoc analysis.
  15. Cloud Run · triage — Pub/Sub push handler. Picks a runbook (cosine over YAML library), asks Gemini to explain, posts to Slack, writes to incidents table.
  16. Cloud Scheduler — cron 0 2 * * * publishes a retrain trigger; Vertex AI Pipelines picks it up.
  17. Cloud Monitoring — uptime check on the triage URL, alert policy on Pub/Sub backlog and BigQuery ingestion lag.
  18. Cloud Logging — every component emits structured JSON; severity-routed sinks feed alert policies.
  19. Grafana — on a small Compute Engine VM, BigQuery datasource. Dashboards: live SLI panel, forecast-vs-actual, drift map.
🔁 End-to-end data flow
  1. VM loadgen publishes raw events & per-metric telemetry to Pub/Sub (~120 msg/s).
  2. GKE telemetry-collector drains the telemetry subscription and streams rows into BigQuery.
  3. Vertex AI victim endpoint serves predictions; its FeatureLogger also publishes telemetry (closes the loop).
  4. GKE forecast-detector queries BigQuery every minute, runs the temporal forecaster, and on predicted SLO breach publishes an incident.
  5. Cloud Run triage consumes the incident: matches a YAML runbook, asks Gemini for context, posts to Slack, writes to BigQuery.
  6. Cloud Scheduler kicks Vertex AI Pipelines nightly to retrain & redeploy the victim / forecaster models.
  7. Cloud Monitoring watches everything; Grafana presents the human-facing dashboards.
