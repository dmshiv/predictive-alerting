📌 Annotated reference — every resource (numbered pins on the diagram)
    🔁 End-to-end data flow
    1. VM loadgen publishes raw events & per-metric telemetry to Pub/Sub (~120 msg/s).
    2. Vertex AI victim endpoint serves predictions & also publishes telemetry (closes the loop).
    3. GKE telemetry-collector drains telemetry → batched BigQuery insert.
    4. GKE forecast-detector reads BigQuery every minute, runs the temporal forecaster, on predicted SLO breach publishes an incident.
    5. Cloud Run triage picks up the incident: matches a runbook · asks Gemini · posts to Slack · writes to BigQuery.
    6. Cloud Scheduler nightly fires the retrain Pub/Sub topic; Vertex AI Pipelines retrains, registers, and redeploys.
    7. Cloud Monitoring watches everything; Grafana presents BQ dashboards.
